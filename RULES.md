# RULES.md — スライド作成の憲法
## ジム・クレイマーのマッドマネー要約チャンネル専用ルール

> このファイルはすべてのスライド作成に必ず適用する。内容の変更は禁止。
> plan.md と組み合わせて使用すること。

---

## 1. 基本仕様

| 項目 | 規定値 |
|------|--------|
| スライド比率 | 16:9（幅 25.4cm × 高さ 14.29cm） |
| スライド枚数 | **最大12枚**（厳守） |
| 1枚目 | 必ず「本日の主要トピック整理」にする。タイトルサムネイルは作らない |
| 時間表示 | スライド本体には記載しない（plan.mdの管理用メモのみ） |

## 2. カラーパレット（変更禁止）

| 役割 | 色名 | カラーコード | 使用箇所 |
|------|------|-------------|---------|
| メインカラー | ダークネイビー | `#1A2B4A` | タイトル・主要テキスト・太い罫線 |
| ポジティブ | ディープグリーン | `#006B3F` | 買い推奨・強み・成長性の強調 |
| ネガティブ | アラートレッド | `#C41E3A` | 売り推奨・マイナス指標・注意喚起 |
| ベース背景 | ホワイト | `#FFFFFF` | スライド背景 |
| セパレーター | ライトグレー | `#E0E0E0` | 区切り線・補足テキスト |
| テキスト（明） | ホワイト | `#FFFFFF` | ネイビー背景上のテキスト |
| テキスト（暗） | ダークネイビー | `#1A2B4A` | 白背景上のテキスト |

### カラー使用ルール
- **ポジティブ（緑）とネガティブ（赤）は意味を持って使う**。装飾目的での使用禁止
- 「買い推奨」「強み」「成長」→ `#006B3F`
- 「売り推奨」「リスク」「マイナス数値」→ `#C41E3A`
- グラデーション・装飾的な色使いは禁止。フラットデザインを維持する

## 3. フォント設定

### フォントファミリー
```
日本語: 'Noto Sans JP', 'Meiryo', 'Hiragino Sans', sans-serif
英数字: 'Calibri', 'Arial', sans-serif
lang属性: ja（PDF出力時の漢字文字化け防止）
```

### フォントサイズ規定

| 要素 | サイズ | ウェイト | 備考 |
|------|--------|---------|------|
| スライドタイトル | 36〜40pt | Bold | 左揃えまたは中央揃え |
| サブタイトル | 20〜22pt | Regular | タイトル直下 |
| カラム・カード見出し | 18〜20pt | Bold | セクション内の小見出し |
| 本文・箇条書き | 15〜17pt | Regular | 読みやすさを最優先 |
| 強調数字（大） | 60〜72pt | Bold | 大数字レイアウト使用時 |
| 強調数字（中） | 36〜48pt | Bold | カード内の数字 |
| 注釈・補足 | 11〜12pt | Regular | スライド下部の小テキスト |

### フォント使用ルール
- **本文テキストは左揃え**（タイトルのみ中央揃えを許可）
- 箇条書きは字下げ1段まで（多段ネストは読みにくいため禁止）
- 太字は見出しと強調箇所のみ。本文中の多用禁止

## 4. レイアウト規定

### 共通マージン・スペーシング
- スライド四辺の余白：**最低1.5cm**
- カラム間・カード間のギャップ：**最低1.0cm**
- テキストブロック間のギャップ：**0.5cm以上**
- 要素を詰め込みすぎない。「余白」も情報のひとつとして扱う

---

### レイアウトパターン定義

plan.mdでは以下のパターン名を指定するだけでよい。

#### [LAYOUT-A] 左右2カラム
```
┌─────────────────────────────────┐
│ タイトル                          │
├────────────────┬────────────────┤
│ 左カラム        │ 右カラム        │
│ （説明・背景）  │ （数値・結論）  │
│                │                │
└────────────────┴────────────────┘
```
- 左右の幅は原則50:50。内容に応じて40:60まで許容
- 左：テキスト説明・背景情報、右：数値・強調・結論が基本配置
- カラム間に`#E0E0E0`の縦罫線を入れる

---

#### [LAYOUT-B] 2×2グリッド（4カード）
```
┌─────────────────────────────────┐
│ タイトル                          │
├────────────────┬────────────────┤
│ カード①        │ カード②        │
├────────────────┼────────────────┤
│ カード③        │ カード④        │
└────────────────┴────────────────┘
```
- 各カードに見出し（18pt Bold）＋本文（15pt）
- カードの背景色：`#F5F7FA`（薄いグレー）、罫線：`#E0E0E0`
- 4カード全て同じサイズで統一

---

#### [LAYOUT-C] 大数字＋説明テキスト
```
┌─────────────────────────────────┐
│ タイトル                          │
├──────────┬──────────────────────┤
│          │ 説明テキスト            │
│  −768    │ ・箇条書き①           │
│  ポイント │ ・箇条書き②           │
│  (72pt)  │ ・箇条書き③           │
└──────────┴──────────────────────┘
```
- 数字は60〜72pt Bold、カラーはコンテキストに応じて緑or赤orネイビー
- 数字の左または中央配置、説明テキストは右または下に配置
- 複数の数字を並べる場合は横並びで統一サイズ

---

#### [LAYOUT-D] アイコン付き縦リスト
```
┌─────────────────────────────────┐
│ タイトル                          │
├─────────────────────────────────┤
│ 🔵 見出し①                       │
│    説明テキスト（15pt）            │
├─────────────────────────────────┤
│ 🔵 見出し②                       │
│    説明テキスト（15pt）            │
├─────────────────────────────────┤
│ 🔵 見出し③                       │
│    説明テキスト（15pt）            │
└─────────────────────────────────┘
```
- アイコンは絵文字または●◆などの記号。画像は使用しない
- 各項目間に`#E0E0E0`の横罫線を入れて区切る
- 1スライドに最大4項目まで（5項目以上はスライドを分割）

---

#### [LAYOUT-E] トピック整理（1枚目専用）
```
┌─────────────────────────────────┐
│ 本日の主要トピック                  │
├─────────────────────────────────┤
│ ① トピックA                       │
│ ② トピックB                       │
│ ③ トピックC     ← 大きめフォントで │
│ ④ トピックD        視認性を確保    │
│ ⑤ トピックE                       │
└─────────────────────────────────┘
```
- 1枚目のみ使用
- フォントサイズは本文より大きめ（18〜20pt）
- 項目数は5〜8個が目安

## 5. 視覚的ルール

### やること
- 各スライドに必ず**1つ以上の視覚的アクセント**を入れる（大数字・カラーボックス・罫線など）
- 強調したい数値は**必ず大きく表示**する（60〜72pt）
- ポジティブ／ネガティブの情報は**必ずカラーで区別**する（緑/赤）
- 「クレイマーの評価」などの引用はボックス囲み＋ネイビー背景＋白テキストで統一

### やってはいけないこと
- グラデーション・シャドウ・装飾的エフェクトの使用
- 1スライドに3種類以上のフォントサイズを混在
- テキストの右揃え（左揃えまたは中央揃えのみ）
- 画像・写真の挿入（アイコン・記号のみ許可）
- 1スライドに情報を詰め込みすぎる（迷ったらスライドを分割）

---

## 6. コンテンツルール

- **「買い推奨」「売り推奨」は必ずカラーで明示**（緑=買い、赤=売り）
- クレイマーの発言を引用する場合は「」で囲みボックス表示
- 数値データは出典・文脈が分かるよう必ずラベルを添える
- 投資情報である旨の注記を最終スライドに必ず入れる：
  ```
  ※本スライドは情報提供のみを目的としており、投資の勧誘を意図するものではありません。
  ```

---

## 7. PDF出力時の注意（日本語文字化け防止）

```css
font-family: 'Noto Sans JP', 'Meiryo', 'Hiragino Sans', sans-serif;
lang="ja"
```

- 必ずHTMLおよびCSS全体に`lang="ja"`を指定する
- 英数字もNoto Sans JPでカバーされるが、Calibriを優先指定すること
- LibreOfficeで変換する場合、フォントの埋め込みを有効にする
